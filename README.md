# Laboratório de Estudos em TypeScript

## Objetivo

Este repositório é destinado exclusivamente a estudos e experimentação com **TypeScript**.  
O foco principal é aprofundar conhecimentos técnicos, validar conceitos e aplicar boas práticas de desenvolvimento.

## Finalidade

- Consolidar fundamentos da linguagem TypeScript.
- Explorar recursos avançados (tipagem avançada, generics, utility types, decorators, etc.).
- Aplicar princípios de orientação a objetos.
- Implementar e praticar os princípios **SOLID**.
- Testar padrões de projeto (Design Patterns).
- Experimentar diferentes abordagens arquiteturais.
- Avaliar organização, modularização e desacoplamento de código.
- Simular cenários reais de desenvolvimento.

## Princípios Aplicados

### SOLID

- **S — Single Responsibility Principle (SRP)**  
  Cada classe ou módulo deve possuir uma única responsabilidade.

- **O — Open/Closed Principle (OCP)**  
  Entidades devem estar abertas para extensão e fechadas para modificação.

- **L — Liskov Substitution Principle (LSP)**  
  Subtipos devem poder substituir seus tipos base sem alterar o comportamento esperado.

- **I — Interface Segregation Principle (ISP)**  
  Interfaces devem ser específicas e não forçar implementações desnecessárias.

- **D — Dependency Inversion Principle (DIP)**  
  Módulos de alto nível não devem depender de módulos de baixo nível, mas de abstrações.
